Seed demo patients

Populate a FHIR test server with rich synthetic patients, then read them in the record reader.

Test servers only. This page writes data (HTTP POST). Only run it against an open test server like HAPI. Never point it at a real CureMD tenant or any server holding real patients — and CureMD's FHIR API is read-only anyway, so writes there would simply fail. All patients below are invented (Synthea-style fakes); no real people.

Target server

    After seeding, open the record reader (same default server) and search Demo to find all of them — each comes with conditions, vitals, labs, medications, allergies, immunizations, procedures, encounters, documents, care plans, and goals.
