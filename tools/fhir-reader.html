FHIR Record Reader

A read-only viewer for any FHIR Release 4 server — this viewer only reads, never writes. (Need test data? The separate demo-patient seeder writes to sandboxes only.)

Text size

Read-only. This tool only issues GET (read/search) requests — it cannot create or change records. It defaults to a public test server with synthetic data. To point it at a real CureMD tenant, set the base URL issued at registration and paste a valid access token below. Never load real patient data you are not authorized to view.

    Built against FHIR R4 / US Core. The generated OpenAPI description of CureMD's API lives alongside this file in knowledge/curemd-fhir-openapi.json. This viewer talks plain FHIR REST, so it works against any conformant R4 server. Unofficial; not affiliated with CureMD. When pointed at CureMD, remember the data refreshes weekly and the rate limit is 20 requests/minute.
